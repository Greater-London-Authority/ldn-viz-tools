# Form element styling

Styling native form elements to be consistent across browser can be a pain.
We provide a basic reset for form styles [via The Tailwind form plugin](https://github.com/tailwindlabs/tailwindcss-forms).
We also provide a set of themed form classes for use in this library and in your projects.

To be flexible when integrating with existing projects we don't generate global form styles, however the theme styles can be applied via class names:

> ðŸ’¡ **Tl;dr**; Apply the `.form-checkbox` class to a native checkbox and it will be themed the same as an import of `<Checkbox />` (though it will only have native functionality).

Here is a complete table of the provided `form-*` classes for reference:

| \`[type='password']\`       | \`form-input\`       |
| \`[type='number']\`         | \`form-input\`       |

| \`[type='datetime-local']\` | \`form-input\`       |
| \`[type='month']\`          | \`form-input\`       |

```html
<!-- apply our theme to any native form element with a single class -->
<input type="text" class="form-input" />

<!-- A full example using checkbox-->
<label class="flex items-center">
	<input class="form-checkbox" type="checkbox" bind:checked />
	<span class="mx-2 form-label">Checkbox Option</span>
</label>

<!-- further customise using utility classes -->
<input type="text" class="form-input text-color-ui-negative w-full" />
```
